# MCU 波形分析（通用版）

## 何时使用
- 分析 Saleae Logic 2 捕获的 `.sal` 波形文件
- 验证 PWM 输出频率、占空比是否与代码配置一致
- 提取 EMS 节律参数（burst ON/OFF、载波频率）
- 对比不同工作模式的波形特征
- 排查输出异常（时序错位、频率偏移、脉宽异常）
- 将实测波形与代码参数交叉验证

## ⚠️ 核心原则

1. **先读代码再分析波形** — 从源代码中提取目标参数（频率、占空比、节律），再与波形对比
2. **区分载波层和节律层** — EMS/PWM 通常有两层：高频载波（~92KHz）+ 低频节律（ms级 ON/OFF）
3. **通道映射靠代码** — 哪个 GPIO 对应哪个信号，以 `02_硬件引脚定义.md` 和 `.c` 代码为准
4. **不同模式不同特征** — 同一产品的不同模式（如 INFUSE/SMOOTH/FIRM）参数差异大，需逐模式分析
5. **关注异常值** — 偶发的超长/超短脉冲通常是 bug 而非随机噪声

## 分析流程

### 第1步：定位波形文件

在项目中搜索波形文件：
1. `*.sal` — Saleae Logic 2 原生格式
2. `waveforms/*/digital_*.bin` — 已导出的二进制通道数据
3. `capture/*_analysis.txt` — 之前的分析摘要
4. `Session_*.sal` — 按模式命名的捕获文件

### 第2步：确定通道映射

从代码和项目文档确认：
- 搜索 `02_硬件引脚定义.md` 查看 GPIO 分配
- 搜索 `.c` 文件中 `PWMxS`、`MOTOR_INIT`、`EMS_Init` 确认输出引脚
- 确认 `.sal` 文件中通道号与信号的对应关系

### 第3步：从代码提取目标参数

搜索源代码中的预期值：

**载波层：**
- PWM 周期寄存器值 → 计算目标频率
- PWM 占空比寄存器值 → 计算目标占空比
- 预分频设置 → 确认时钟源

**节律层：**
- `ems_on_ms` / `ems_off_ms` 等节律参数
- `EMS_OFF_*[]` 节律表数组
- `Motor_SetMode()` 中的马达行为

### 第4步：解析波形数据

#### 如果已有 Python 脚本
项目中有 `analyze_final.py`、`analyze_waveform.py` 等脚本时，直接运行或参考其逻辑。

#### 如果是 .sal 文件
搜索或创建 Python 解析脚本，核心逻辑：
```python
# 读取二进制通道数据
# 检测信号跳变，计算每段高/低电平宽度
# 分类：短脉冲(<50us) = 载波，长间隔(>1ms) = 节律
# 统计载波频率、占空比、burst ON/OFF 时长
```

### 第5步：生成分析报告

对每个通道和模式，输出：
```
## 模式: XXX
### 通道X: EMS
- 载波频率: XX KHz (目标: XX KHz) → [✅/⚠️/🔴]
- 占空比: XX% (目标: XX%)
- Burst ON: X.X ms
- Burst OFF: {序列} (对比代码节律表)
- 节律频率: X.X Hz

### 通道Y: Motor
- 周期: XX ms (频率: X.X Hz)
- 占空比: XX%
- ON/OFF 序列: {序列}

### 综合对比
| 参数 | 目标值 | 实测值 | 偏差 | 状态 |
|------|--------|--------|------|------|
| 载波频率 | 92KHz | 91.8KHz | -0.2% | ✅ |
| ... | ... | ... | ... | ... |
```

### 第6步：交叉验证

将实测值与代码定义对比：
- [ ] 载波频率误差是否在 ±5% 内？
- [ ] 占空比偏差是否在 ±3% 内？
- [ ] Burst ON 时间是否匹配 `ems_on_ms`？
- [ ] Burst OFF 序列是否匹配节律表？
- [ ] 异常脉冲是否与代码中的特殊处理对应？

## 常见波形分析场景

### EMS 输出分析
```
┌─ Burst ON ─┐   ┌─ OFF ─┐   ┌─ ON ─┐   ┌── OFF ──┐
 ▓▓▓▓▓▓▓▓▓▓▓▓               ▓▓▓▓▓▓▓▓
  ↑ 载波 92KHz ↑              ↑
  PWM 连续输出                无输出

分析要点：
- 载波周期 ~10.9us → 频率 ~92KHz
- Burst ON: 通常 2-5ms
- Burst OFF: 依节律表变化 (2ms~471ms)
- 关注异常：OFF 时间与代码节律表不一致
```

### 马达振动分析
```
┌─ VIB ON ──┐  ┌─ OFF ─┐  ┌─ ON ─┐
 ████████████              ████████
 
分析要点：
- INFUSE 模式下振动频率较高 (~5.5Hz)
- FIRM/SMOOTH 下马达随 EMS burst 同步
- 关机时 500ms 长振动一次
```

### 多通道联合分析
```
Channel 0 (MOT):  ____█▁▁▁____█▁▁▁____
Channel 1 (EMS):  _▓▓▓____▓▓▓____▓▓▓___
                    ↑ 同步 ↑ 同步 ↑

分析要点：
- EMS burst ON 时马达是否同步开启
- 时序是否有偏移/延迟
```
